Interface IDbContext

Interface for managing connection strings and connections. Helps keep connections alive between database and service calls, mainly to prevent a transaction from implicitly being elevated to a Distributed Transaction when not necessary.

Remarks

Database connections opened from this instance will only be closed/returned to the pool when this instance is closed/disposed of.

Remarks

Since a database connection can only have one transaction context during its life-time (that is, until it is closed/returned to the pool), it will be necessary to call OpenNewConnection(string) instead of OpenConnection(String) if we want to use a connection with the same name from the same IDbContext instance inside a new transaction.
In other words, we cannot have multiple transactions on the same connection before it is closed, which means, before this IDbContext instance is closed.
